# MRT 코드 리뷰

PR의 코드 변경사항을 마이리얼트립 개발 기준에 맞게 리뷰한다.

## 리뷰 순서

1. **보안** - SQL 인젝션, XSS, 인증/인가 누락, 민감 정보 노출
2. **성능** - N+1 쿼리, 불필요한 루프, 캐싱 기회
3. **정확성** - 엣지 케이스, 에러 처리, 비즈니스 로직 오류
4. **가독성** - 변수명, 함수 분리, 주석 필요 여부

## 출력 형식

각 항목은 아래 형식으로 작성한다:

```
### [심각도] 파일명:줄번호

**문제**: 무엇이 문제인가
**이유**: 왜 문제인가
**제안**: 어떻게 고치면 되는가
```

심각도: 🔴 Critical / 🟡 Warning / 🔵 Suggestion

## 마무리

리뷰 마지막에는 다음을 포함한다:
- **총평**: 전체적인 코드 품질 평가 (1-2문장)
- **칭찬할 점**: 잘 된 부분 반드시 1개 이상
- **머지 권고**: Approve / Request Changes / Needs Discussion
