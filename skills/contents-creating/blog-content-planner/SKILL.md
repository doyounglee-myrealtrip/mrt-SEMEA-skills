# Phase 1: 블로그 신선도 점검 (audit 모드)

> ⚠️ **네이버 블로그 접근 제한**: `blog.naver.com`은 크롤러가 완전히 차단되어 WebFetch로 접근 불가합니다. Google도 네이버 블로그를 인덱싱하지 않습니다.
>
> **대안**: 팀원이 네이버 블로그 내부 검색 결과를 수동 수집한 `검색어.txt` 파일을 제공하면, 해당 파일을 1차 데이터 소스로 사용합니다. 파일이 없으면 아래 WebSearch 기반 간접 탐지를 실행합니다.

### 수동 데이터 모드 (`검색어.txt` 제공 시)

1. 사용자가 제공한 `검색어.txt` 파일을 읽습니다
2. 각 도시/테마별로 가장 최근 블로그 글의 제목·발행일을 추출합니다
3. 발행일 기준으로 경과일을 계산하고 STEP 1-2의 신선도 등급을 매깁니다
4. WebSearch는 보조 확인용으로만 사용합니다

- 한국인 여행자 후기 우선 인용
- 구체적 수치(금액·시간·거리) 포함
- **팁은 항상 "이렇게 하면 더 좋다" 형태로 작성**
- 부정적 경험은 "문제 → 해결책 → CTA" 형태로만 작성
- 해결책 없는 불편함·위험·단점은 포함하지 않음

### 8. FAQ (4~6개 — 롱테일 질문으로)
- 실제로 AI 검색창에 칠 법한 구체적·상황적 질문으로 작성
- "어떻게 하면 잘 즐길 수 있는가" 형태 위주
- 각 답변은 40~80자 내외로 간결하게

### 9. E-E-A-T 신뢰도 (직접 경험이 있을 때만)
- 별도 고지문 형태로 쓰지 말 것
- 직접 경험은 본문 흐름 안에 자연스럽게 녹이기
- 직접 경험이 없으면 이 항목 전체 생략

### 10. 마무리
- 본문에서 다룬 구체적인 장소·경험을 1~2개 언급하며 마무리
- CTA는 템플릿 형태 금지, 앞 문장 흐름에서 자연스럽게
- 마이리얼트립 상품 간 직접 비교 금지
- 마무리 분량은 3~5줄

## STEP 4: 문체 규칙

- 친근하고 가볍게, 여행 유경험자 친구가 조언하는 느낌
- "~예요", "~거예요", "~해요" 톤 선호
- 단, 각 문단의 첫 문장(결론)은 단정적으로 유지
- "~인 것 같아요", "~일 수도 있어요" 표현 금지
- 문장은 2~3줄 이내로 짧게 끊기

## STEP 5: 해시태그 생성 (필수)

블로그 글 완성 후, 네이버 블로그용 해시태그를 **10~15개** 생성합니다.

해시태그 규칙:
- 도시명 + 주제 조합 (예: #그라나다여행, #알함브라궁전예약)
- 일반 여행 태그 (예: #유럽여행, #스페인여행)
- 롱테일 태그 (예: #그라나다알함브라예약방법, #2026유럽여행)
- 마이리얼트립 태그 (예: #마이리얼트립, #마이리얼트립투어)
- 계절/시기 태그 (예: #6월유럽여행, #여름스페인)

출력 형식:
```
🏷️ 추천 해시태그 (10~15개)
#그라나다여행 #알함브라궁전 #알함브라예약 #스페인여행 #유럽여행 #그라나다맛집 #안달루시아 #마이리얼트립 #2026유럽 #그라나다당일치기 #스페인남부여행 #알함브라궁전예약방법
```

## STEP 7: Google Docs 초안 저장 + 캔버스 갱신

### 7-1: Google Docs에 초안 저장

Google Drive MCP를 사용하여 블로그 초안을 Google Docs 문서로 생성합니다.

1. Google Docs 새 문서를 생성합니다
   - 제목: `[SEMEA 블로그] <도시명> — <주제> (YYYY-MM-DD)`
   - 본문: 완성된 블로그 글 전문 + 해시태그 + 이미지 가이드(있는 경우)
2. 생성된 문서의 공유 링크를 확보합니다

### 7-2: Slack 채널에 결과 공유

Slack 채널 `C0B2LB8868Z`의 스레드(또는 새 메시지)에 아래를 전달합니다:

```
✅ 블로그 초안이 완성됐어요!

📝 <도시명> — <주제>
📄 Google Docs: <공유 링크>

검토 후 네이버 블로그에 발행해 주세요.
해시태그도 문서 하단에 포함되어 있어요 🏷️
```

### 7-3: 캔버스 작성 이력 추가

캔버스 `F0B9Q17NE3Y`의 **✅ 작성 완료 이력** 섹션에 새 행을 추가합니다.

1. `slack_read_canvas`로 캔버스의 현재 섹션 ID를 확인합니다
2. 작성 이력 테이블 섹션을 찾아 `slack_update_canvas`로 행을 추가합니다:

추가할 행:
```
| YYYY-MM-DD | <도시명> | <주제(짧게)> | <작성자> | <Google Docs 링크> |
```

### 7-4: 신선도 등급 갱신

같은 캔버스의 **도시별 신선도** 테이블에서 해당 도시/테마의 등급을 갱신합니다.

1. 해당 도시의 행을 찾습니다
2. `등급`을 🟢(오늘 작성)로, `최근 글 제목`을 새 블로그 제목으로, `발행일`을 오늘 날짜로, `경과일`을 0일로 업데이트합니다

> ⚠️ `slack_update_canvas`에서 `action=replace`를 `section_id` 없이 사용하면 캔버스 전체가 덮어씌워집니다. 반드시 `slack_read_canvas`로 section_id를 확인한 후 해당 섹션만 업데이트하세요.

### 터미널에서 호출된 경우

Google Docs 생성 + 캔버스 갱신은 동일하게 수행하고, 블로그 전문은 터미널에도 함께 출력합니다.
